# Solen Protocol Specifications

Canonical protocol specifications for the Solen network. These define the terms, state model, and API surface that all implementations must conform to.

## Specs

| Spec | Status | Description |
|------|--------|-------------|
| [001 - Terminology](./001-terminology.md) | Draft | Definitions of key protocol terms |
| [002 - State Model](./002-state-model.md) | Draft | Canonical state objects and their fields |
| [003 - RPC API](./003-rpc-api.md) | Draft | JSON-RPC method signatures |

## Adding a Spec

New specs should follow the numbering convention (`NNN-title.md`) and include a status field (Draft, Review, Final). Specs are implementation-agnostic and should be suitable for building alternative clients.
